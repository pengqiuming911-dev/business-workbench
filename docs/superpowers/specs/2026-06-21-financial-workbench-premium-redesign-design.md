## 1. 总体判断

当前工作台的主要问题不是功能缺失，而是视觉语言和交互层级失控：

- 主任务区没有稳定占据最高优先级，`AgentDrawer` 默认展开并覆盖内容，持续打断使用。
- 全站已有基础 token 和卡片体系，但仍停留在通用 SaaS 后台风格，金融专业感不足。
- 信息密度高的核心页已经具备业务深度，但筛选区、表格语义、数字排版和固定列策略还不够克制。
- 低密度页面的留白没有经过版心控制，导致“空”和“未完成”观感接近。

目标不是“做得更花”，而是把产品气质统一到以下关键词：

- 克制
- 冷静
- 对齐
- 语义明确
- 辅助不抢主任务

## 2. 当前代码中的关键问题

### 2.1 智能体抽屉喧宾夺主

`frontend/components/WorkbenchLayout.vue`

- `drawerOpen` 当前默认值为 `true`。
- 页面右下角已经同时存在 `agent-fab`，与默认展开抽屉重复。
- 抽屉现在是 fixed overlay，打开后直接压住主内容，不属于工作台级的高质量布局策略。

`frontend/components/AgentDrawer.vue`

- 宽度固定 `400px`，没有推开主内容的布局联动。
- 没有记忆用户上次开合状态。
- 视觉上使用绿色强调，和全站金融工作台取向不一致，更像聊天插件。

### 2.2 设计系统存在，但风格还不够专业金融

`frontend/assets/main.css`

- 已有颜色、阴影、圆角、表格、按钮、标签等基础 token，这是好基础。
- 但主色仍偏互联网工具蓝：`--brand: #2962ff`。
- 背景、卡片、阴影层级偏轻，无法形成“冷灰画布 + 白卡浮起”的秩序感。
- 卡片圆角、按钮圆角、badge 圆角层级还不够稳定。
- 数字虽然部分使用 `--font-mono`，但未形成“所有金额/比例/指数统一等宽右对齐”的严格规范。

### 2.3 核心页的问题主要集中在筛选与表格秩序

`frontend/views/ProductAnalysis.vue`

- 主筛选和高级筛选已经分层，这是正确方向。
- 但筛选器仍像字段堆叠，没有“当前过滤态回显”。
- 表格头部颜色偏暖黄，和金融工作台主色体系不一致。
- 左侧固定列已经存在，但缺少右侧操作区固定、表头排序、过滤结果总结。

`frontend/views/RebatePending.vue`

- 筛选条件过多，第一屏认知负担很高。
- 多组色块表头表达了分组，但颜色数量过多，显得嘈杂。
- 操作按钮密集且语义同级，缺少“主操作 / 次操作 / 行内操作”的层级划分。

### 2.4 低密度页面缺少版心与空状态设计

`frontend/views/PushSettings.vue`

- 表单宽度跟随容器拉满，长字段直接横向铺开，质感偏弱。
- 状态信息与配置区域平铺，没有明显主次。

`frontend/views/UserProfile.vue`

- 页面标题是“用户画像”，但侧边导航文案还是“账户”。
- 这会造成信息架构错位，降低专业度。

## 3. 重构目标

### 3.1 产品气质目标

- 从“通用后台”升级为“专业金融运营工作台”。
- 让高密度数据场景看起来有秩序，而不是靠减内容制造高级感。
- 让辅助能力退居二级，不干扰主工作流。

### 3.2 交互目标

- 默认进入任何页面时，主内容完整可见。
- 筛选应遵循“常用条件常驻，次要条件折叠”的原则。
- 表格的扫读效率优先于装饰性。
- 所有状态、金额、比例和结构类型都应具备稳定的语义表达。

### 3.3 实施目标

- 先改框架层，不先逐页堆样式。
- 先定义 token 和布局规则，再改核心页面。
- 优先复用现有组件，不引入新的大型 UI 框架。

## 4. 设计系统方向

### 4.1 配色

建议改为更稳重的冷灰 + 深蓝体系：

- 页面背景：`#F5F7FA` 或 `#F6F8FB`
- 一级卡片：`#FFFFFF`
- 二级浅底：`#EEF2F7`
- 主色：`#1F3A8A`
- 主色 hover / 强调：`#2C5CE0`
- 文字主色：`#0F172A`
- 次级文字：`#475569`
- 弱化文字：`#94A3B8`
- 分隔线：`#E2E8F0`

语义色要求：

- 上涨 / 正向：统一为低荧光红
- 下跌 / 风险：统一为低荧光绿或风险红，必须先统一业务约定
- 警示：统一琥珀色
- 中性状态：统一蓝灰

要求：

- 不再在同一张表头中铺太多高识别度底色。
- 用少量语义色，更多依赖边框、明度、字重和标签。

### 4.2 字体与数字

- 正文延续中文无衬线，但减少 `Inter` 的存在感，优先中文金融后台可读性。
- 所有金额、比例、指数、日期计数统一使用 `font-variant-numeric: tabular-nums`。
- 金额列、比例列、份额列统一右对齐。
- KPI、指数卡片、表格金额禁止混用比例字体和普通字体。

### 4.3 圆角与阴影

- 卡片圆角：`12px`
- 输入框 / 按钮圆角：`8px`
- badge / tag 圆角：`999px` 或 `6px`，二选一，不混用
- 卡片阴影：只保留两级
- 悬浮 / 抽屉 / 对话框：使用更明显但不发灰的深色阴影

### 4.4 间距

统一 4 / 8 栅格：

- 页面主区横向内边距：`32px`
- 卡片内边距：`20px` 或 `24px`
- 模块间距：`16px / 24px`
- 筛选器行距：`12px`
- 表单控件高度：统一到 `36px` 或 `40px`

## 5. 关键改造方案

### 5.1 P0：智能体面板重构

涉及：

- `frontend/components/WorkbenchLayout.vue`
- `frontend/components/AgentDrawer.vue`

改造要求：

- 默认收起，`drawerOpen` 初始值改为 `false`。
- 使用 `localStorage` 记住用户上次开合状态。
- 保留右下角 FAB 作为唯一入口，移除“默认展开 + FAB 并存”的重复关系。
- 桌面端展开时优先采用“推开主内容”的 docked drawer，而不是覆盖内容。
- 移动端保留 overlay 模式。
- 抽屉宽度支持 `360px - 440px` 的可调区间，初版可先固定，再加拖拽。

视觉要求：

- 抽屉品牌色从绿色切换到全站主色体系。
- 欢迎区和建议按钮降噪，避免像独立聊天产品。

验收标准：

- 任意页面首次进入时，表格和筛选区不被右侧面板遮挡。
- 关闭抽屉后刷新页面，状态保持关闭。
- 打开抽屉后主内容宽度缩小但不被覆盖。

### 5.2 P0：全站 token 收口

涉及：

- `frontend/assets/main.css`
- `frontend/components/PanelCard.vue`
- `frontend/components/SidebarNav.vue`

改造要求：

- 重定义 `:root` 下颜色、阴影、圆角、文字层级 token。
- `PanelCard` 统一为白卡浮层，不再让页面自己重复定义卡片观感。
- 侧边栏从纯白工具栏改为更轻的浅灰面板，提升框架秩序感。
- `tab-bar`、`filter-bar`、`data-table`、`badge`、`btn` 统一跟随新 token。

验收标准：

- 不同页面卡片边框、圆角、阴影一致。
- 表格 hover、筛选区、侧边导航、按钮状态统一。
- 页面整体视觉重心从“亮蓝按钮”变为“数据内容”。

### 5.3 P1：产品分析页重构

涉及：

- `frontend/views/ProductAnalysis.vue`

改造要求：

- 将主筛选保留为 4 项以内，例如：申购日期、完结日期、持有状态、管理人。
- 其余字段进入“高级筛选”折叠区。
- 在表格上方新增“已选条件回显区”，以 chips 展示过滤条件。
- 新增结果摘要区：总条数、当前筛选数、关键汇总指标。
- 表头从暖黄色改为中性浅灰或浅蓝灰。
- 左固定列保留，增加右侧操作或状态固定区的预留规范。
- 数字列全部采用统一的 `num` 类。
- 结构类型、持有状态改为语义 tag，而不是纯文本。

验收标准：

- 首屏筛选区明显更短。
- 用户不展开高级筛选也能完成大部分查询。
- 横向滚动时固定列和表头表现稳定。

### 5.4 P1：返费页重构

涉及：

- `frontend/views/RebatePending.vue`
- `frontend/views/RebateCompleted.vue`

改造要求：

- 第一层只保留高频筛选：订单号、客户姓名、返费对象、是否可返。
- 分类多选、校验状态、航班编号、产品名等进入高级筛选抽屉或折叠区。
- 已选条件必须回显。
- 表头分组保留，但压缩颜色数量，采用 1 个主中性色 + 少量语义辅助色。
- 行内“返申购费 / 返管理费 / 返业绩报酬”按钮需要弱化为次级操作。
- 批量操作区做成独立工具条，避免和筛选区混在一起。

验收标准：

- 首屏不再出现十多个筛选项同时铺满。
- 分组表头依然可读，但整体更安静。
- 行内操作不会成为页面视觉焦点。

### 5.5 P1：总览页精修

涉及：

- `frontend/views/Dashboard.vue`

改造要求：

- 压缩 hero 区无效留白。
- 增大指数迷你图占比，减小“装饰性容器”的存在感。
- KPI 卡片增加环比、趋势或状态说明，避免只剩大数字。
- 同步状态区域升级为更清晰的状态卡。
- “智能助手”按钮若保留，应明确为次级入口。

验收标准：

- 总览页不只“看起来整洁”，而是有稳定的信息节奏。
- 指数卡片更像专业行情摘要，而不是通用仪表盘卡片。

### 5.6 P2：低密度页面补完成度

涉及：

- `frontend/views/PushSettings.vue`
- `frontend/views/ProductReport.vue`
- `frontend/views/UserProfile.vue`

改造要求：

- `PushSettings` 收窄版心，配置表单和状态卡分栏或纵向分组。
- `Webhook URL` 提供复制按钮和固定宽度展示策略。
- `ProductReport` 默认选中当月，空状态使用插画或结构化提示。
- `UserProfile` 菜单改名为“客户画像”，若确有账户设置，则新增独立页面。

验收标准：

- 空白区域变成“有结构的留白”，不再像未完成页面。
- 导航命名与页面职责一致。

3. 统一 `PanelCard`、`SidebarNav`、`btn`、`filter-bar`、`data-table`

### 阶段二：核心业务页

1. `ProductAnalysis.vue`
2. `RebatePending.vue`
3. `RebateCompleted.vue`

### 阶段三：收尾页与命名修正

1. `Dashboard.vue`

## 7. 不建议的做法

- 不建议先逐页单独润色颜色和圆角，否则很快再次失控。
- 不建议为了“高级感”大幅降低信息密度，金融后台的价值就是高密度但可读。
- 不建议引入过多渐变、玻璃拟态、彩色表头和重动画。
- 不建议让智能体继续占据固定右栏且默认展开。

## 8. 三个最值得立即做的改动

1. 收起智能体面板，改为记忆状态且不遮挡主内容。
2. 收口全站 token，建立冷灰背景、白卡浮层、深蓝主色、统一等宽数字。
3. 重做产品分析与返费页的筛选区和表格语义层级。

## 9. 验收清单

- 任意页面首次进入时，主数据区完整可见。
- 全站金额、比例、指数、数量采用统一数字排版。
- 表格页首屏筛选数量明显下降，次要项默认折叠。
- 标签、按钮、卡片、导航、抽屉遵守同一套视觉 token。
- “账户 / 用户画像”命名不再冲突。
- 页面看起来更像金融运营台，而不是默认后台模板。
