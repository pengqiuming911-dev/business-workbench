# 飞书集成开发记录

## 项目背景

业务工作台（business-workbench）是一个 Vue 3 前端 + Node.js Express 后端的数据分析平台。
目标是通过飞书开放平台 API，读取用户飞书云盘中的 Excel 文件，用于后续数据处理分析。

---

## 整体架构

```
浏览器（Vue 前端 :5173）
    ↕  /api/* 代理
Node.js 后端（Express :3001）
    ↕  HTTPS
飞书开放平台 API
    ↕  同步写入
SQLite 本地数据库（server/data.sqlite）
```

- 前端通过 Vite 代理将 `/api/*` 请求转发到 `localhost:3001`
- 后端负责 OAuth 授权流程、持有 user_access_token、代理飞书 API 请求
- 飞书数据通过「数据准备」页面手动触发同步，写入本地 SQLite
- 前端功能页面直接查询本地 SQLite，不再每次调用飞书 API

## 一、飞书应用创建步骤

1. 访问 [open.feishu.cn](https://open.feishu.cn) 并登录
2. 进入**开发者后台** → **创建应用** → 选择「自建应用」
3. 创建完成后进入**凭证与基础信息**，记录：
   - `App ID`（格式：`cli_xxxxxxxxxx`）
   - `App Secret`

### 权限配置

进入**权限管理** → 搜索 `drive`，开通以下权限：

| 权限标识 | 说明 |
|---|---|
| `drive:drive` | 读写云盘文件 |
| `drive:file` | 读取文件内容 |
| `drive:export:readonly` | 导出云盘文件（实测共享文档无效，保留备用） |
| `space:document:retrieve` | 访问共享空间文档 |

> **重要：授权方式必须设为「用户授权」**
>
> 每个权限旁边有「授权方式」列，必须是「**用户授权**」而不是「应用授权」。
> 如果是「应用授权」，OAuth 授权时飞书会悄悄忽略这个 scope，导致 token 没有云盘权限。
> 确认后需要重新发布版本才能生效。

### 安全设置

进入**安全设置** → **重定向 URL**，添加：

```
http://localhost:3001/api/auth/callback
```

### 发布应用

进入**版本管理与发布**：
- 个人/测试用途：选「内测」→ 添加自己的飞书账号为内测用户，无需管理员审核，立即生效
- 正式上线：创建版本申请线上发布，需要企业管理员审核

### API 端点

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/auth/login` | 返回飞书 OAuth 授权 URL |
| GET | `/api/auth/callback` | 飞书回调，用 code 换 token |
| GET | `/api/auth/status` | 查询当前授权状态 |
| DELETE | `/api/auth/logout` | 清除本地 token |
| GET | `/api/drive/files` | 列出云盘文件，支持 `?folder_token=` |

| GET | `/api/db/sync-status` | 查询最近一次同步时间和行数 |

| GET | `/api/debug/token` | 调试：查看当前 token 用户信息和已授权 scope |

## 五、OAuth 授权流程

```
1. 前端调用 GET /api/auth/login
      ↓ 返回 { url: "https://open.feishu.cn/..." }

2. 前端将浏览器重定向到该 URL
      ↓ 用户在飞书页面登录并点击「同意授权」

3. 飞书回调 GET /api/auth/callback?code=xxx
      ↓ 后端用 code 换取 user_access_token（两步）
        a. POST /auth/v3/app_access_token/internal → app_access_token
        b. POST /authen/v1/oidc/access_token → user_access_token

4. user_access_token 存入后端内存
      ↓ 后端重定向到前端 http://localhost:5173?auth=success

5. 前端检测到 auth=success，调用 /api/auth/status 确认授权状态
```

## 六、验证步骤

### 验证后端启动正常

```bash
curl http://localhost:3001/api/auth/status
# 期望返回: {"authorized":false}
```

### 验证授权 URL 生成正常

浏览器访问：
```
http://localhost:3001/api/auth/login
```

期望返回的 JSON 中，`url` 字段包含真实的 `app_id` 和 `redirect_uri`（均不为 `undefined`）：

```json
{
  "url": "https://open.feishu.cn/open-apis/authen/v1/authorize?app_id=cli_xxx&redirect_uri=http%3A%2F%2Flocalhost%3A3001%2Fapi%2Fauth%2Fcallback&scope=drive:drive:readonly%20drive:file&response_type=code"
}
```

### 常见问题排查

| 现象 | 原因 | 解决 |
|---|---|---|
| `app_id=undefined` | `.env` 文件不存在或未重启后端 | 创建 `.env` 后重启 `npm run dev` |
| `redirect_uri=undefined` | `.env` 里缺少 `FEISHU_REDIRECT_URI` | 补全 `.env` 后重启 |
| 授权后跳回 `auth=error` | 飞书后台未配置重定向 URL 或应用未发布 | 检查安全设置和发布状态 |
| 前端调用 `/api/` 报 404 | 后端未启动，或 vite 代理未配置 | 确认后端在 3001 端口运行 |
| `/api/drive/files` 报 99991679 | token 没有云盘权限 | 见下方「drive 权限不生效」排查 |

### drive 权限不生效排查

**现象：** 授权成功（`auth=success`），但 `/api/drive/files` 返回 99991679 权限错误，后端日志显示 `实际授权的 scope: auth:user.id:read`（没有 drive 相关 scope）。

**原因：** 飞书权限管理里，drive 权限的「授权方式」是「应用授权」而非「用户授权」，OAuth 时飞书会自动忽略这些 scope。

**解决步骤：**

1. 飞书开发者后台 → **权限管理** → 找到 `drive:drive`、`drive:drive:readonly`、`drive:file`
2. 将每个权限的「授权方式」改为「**用户授权**」
3. **版本管理与发布** → 重新发布版本（权限变更必须发布才生效）
4. 后端重启：`Ctrl+C` → `npm run dev`
5. 重新授权：访问 `http://localhost:3001/api/auth/login`，打开返回的 `url`，同意授权
6. 确认后端日志输出包含 drive scope：
   ```
   实际授权的 scope： drive:drive drive:file ...
   ```
7. 再测试 `http://localhost:3001/api/drive/files`，应返回文件列表

## 九、SQLite 数据同步机制

### 数据库结构（server/db.js）

```sql
-- 产品表
CREATE TABLE IF NOT EXISTS products (
  id TEXT PRIMARY KEY,        -- 航班编号
  name TEXT,                  -- 产品名称
  is_main INTEGER,            -- 是否主产品（1=是，0=否）
  issue_date TEXT,            -- 认购日（YYYY-MM-DD）
  complete_date TEXT,         -- 完结时间（YYYY-MM-DD）
  subscribe_amount REAL,      -- 认购金额
  outstanding_amount REAL,    -- 存续金额
  raw TEXT                    -- 原始行数据（JSON）
);

-- 同步日志表
CREATE TABLE IF NOT EXISTS sync_log (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  synced_at TEXT,             -- 同步时间（ISO 8601）
  row_count INTEGER           -- 同步行数
);
```

### 数据源配置（server/index.js）

```js
const SHEET_TOKEN = 'X1mQbEMqroPjDtxnZx2cO8bpn8f'   // 航班服务交易总表

const SHEETS_META = {
  产品表: { id: '3JiyjX', rows: 134, cols: 58 },
}
```

> sheet token 和 sheet id 从飞书 URL 中获取：
> `https://kcngap16uccc.feishu.cn/sheets/{sheet_token}?sheet={sheet_id}`

### 同步逻辑

1. 用飞书 Sheets v2 API 分批读取产品表（每批 500 行，按精确行列范围避免超出 10MB 限制）
2. Excel 日期序列号转换为 `YYYY-MM-DD` 字符串（`(num - 25569) * 86400 * 1000`）
3. 与 DB 现有数据对比：
   - 飞书有、DB 没有 → **插入**
   - 飞书有、DB 也有 → **覆盖**（`INSERT OR REPLACE`）
   - 飞书没有、DB 有 → **删除**

### 注意事项

- 飞书 Sheets API 返回的日期是 Excel 序列数（如 `45037.0`），不是日期字符串，需要手动转换
- 飞书 Export API 对共享文档无效（`job_status=2`），改用 Sheets values API 直接读取单元格数据
- Sheets values API 请求范围过大会报 `90221 data exceeded 10485760 bytes`，需通过 `/api/debug/sheet-meta` 获取精确行列数后再限定范围
- `valueRenderOption`、`dateTimeRenderOption` 等参数在 v2 API 无效，会报 `90204 unknown option`，不要传这些参数

## 十、产品完结与发行报告逻辑

页面：`src/views/ProductCompletion.vue`

### 计算规则

| 指标 | 定义 |
|---|---|
| 完结产品数量 | `complete_date` 在所选日期范围内的产品数（按航班编号去重） |
| 完结产品金额 | 上述产品的 `subscribe_amount` 之和（万元） |
| 发行数量 | `issue_date`（认购日）在所选日期范围内的产品数（按航班编号去重） |
| 发行金额 | 上述产品的 `subscribe_amount` 之和（万元） |
| 净增金额 | 发行金额 − 完结产品金额 |

- 日期为空的记录忽略不计
- 不区分主产品/子产品，不同航班编号视为不同产品
- 点击表格中的数量数字可展开查看具体航班编号列表

### 常见问题

**Q：选 2026-03 却出现 2025 年数据？**

原因是 SQL 查询用 `complete_date OR issue_date` 两个条件，返回了"2025 年发行、2026-03 完结"的产品，前端 `compute()` 再按实际日期分组，导致 2025 的发行月份也出现在表格里。

解决：`compute()` 里对每个事件做二次日期过滤，只有日期落在所选范围内才计入。

**Q：某产品 complete_date 不应该有值，但 DB 里有？**

飞书表格该单元格可能有隐藏内容（空格、0 等），被转换成了日期。需要去飞书原表确认，清空后重新同步。
